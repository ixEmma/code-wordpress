# Header Build Lessons

## Scope

This file records header-specific lessons for the 1st Providence rebuild. It does not modify reusable `codex-skills` and is not a universal Elementor standard.

## Lessons

1. Header work is allowed here because the user explicitly requested a separate header rebuild.
2. The updated screenshot shows a much taller desktop header than the earlier full-page reference.
3. The logo should be treated as a major desktop object, around 290px wide, not as a compact nav logo.
4. The desktop header should use a wide inner container around 1700px so the nav and CTAs sit like the screenshot.
5. Programs is visually a simple vertical dropdown panel, so a styled Elementor Pro Nav Menu dropdown is an acceptable import-safe fallback.
6. Do not hardcode menu links as text widgets; preserve WordPress menu editability.
7. If the installed Elementor Pro supports the newer Menu/Mega Menu widget, that can replace the Nav Menu widget after import for richer editing.
8. CTA buttons must keep the project button lock from `AGENTS.md`.
9. Mobile should prioritize logo plus burger; duplicate Payment and Contact Us into the WordPress menu if separate mobile buttons feel crowded.
10. Keep header, homepage body, footer, reference exports, and reusable skills separated.
