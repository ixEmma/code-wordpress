# Distinctive Frontend Design Skill

## Purpose

Use this skill when designing or reviewing a homepage, landing page, website section, interface, or visual system.

This skill helps avoid generic AI-looking design. The goal is to create distinctive, intentional, production-grade interfaces with strong visual direction, clean hierarchy, and real design decisions.

For this project, this skill must support Elementor and WordPress work. Do not let creative design choices break Elementor editability, global styles, responsiveness, accessibility, or import compatibility.

## Core Principle

Choose a clear design direction before building.

A strong design can be minimal, bold, editorial, luxury, corporate, healthcare-focused, local-business friendly, modern SaaS, organic, industrial, playful, or refined. The key is intentionality.

Do not create a design that feels like a generic template.

## Before Designing

Analyze the brief or reference first:

1. What business or product is this for?
2. Who is the target audience?
3. What should the visitor understand first?
4. What action should the visitor take?
5. What visual tone fits the brand?
6. What makes this page or section memorable?
7. What should remain simple for Elementor editing?
8. What styles should be globalized?

Do not start from random colors, random gradients, random cards, or random effects.

## Aesthetic Direction Options

Pick an aesthetic direction that fits the project. Examples:

1. Brutally minimal.
2. Luxury and refined.
3. Editorial or magazine-like.
4. Corporate and trustworthy.
5. Healthcare clean and reassuring.
6. Local business warm and practical.
7. Bold campaign landing page.
8. Organic and natural.
9. Industrial and utilitarian.
10. Art deco or geometric.
11. Soft and calm.
12. Premium agency style.
13. Modern SaaS.
14. Product-focused ecommerce.

The selected direction should control typography, spacing, color, imagery, layout, and motion.

## Anti-AI-Slop Rules

Avoid generic AI design patterns such as:

1. Purple-blue gradients used without brand reason.
2. Random glassmorphism cards.
3. Overused bento grids when they do not fit the content.
4. Fake dashboard mockups with no purpose.
5. Too many glows.
6. Random floating shapes.
7. Same-looking rounded cards everywhere.
8. Overcrowded hero sections.
9. Weak typography hierarchy.
10. Generic stock-photo placement.
11. Predictable centered layout for every section.
12. Repeating the same section rhythm across the full page.

Do not make every project look like a SaaS landing page unless the brand actually requires it.

## Elementor Compatibility Rule

When this skill is used for Elementor or WordPress:

1. Use Elementor Containers.
2. Use native Elementor widgets where possible.
3. Keep all content editable.
4. Use global colors.
5. Use global typography.
6. Use global button styles.
7. Avoid hardcoded HTML unless needed.
8. Avoid complex custom animation that Elementor cannot manage.
9. Avoid effects that require fragile custom JavaScript unless explicitly requested.
10. Keep the final build maintainable inside Elementor.

Creative design must not reduce editability.

## Typography Guidance

Typography should carry the design.

Use typography intentionally:

1. Choose heading style based on the brand tone.
2. Use strong hierarchy between hero heading, section heading, card heading, and body text.
3. Keep paragraph line lengths readable.
4. Use fewer type styles, but use them consistently.
5. Avoid default-looking typography unless the reference requires a plain corporate style.
6. Use global typography tokens for repeated styles.

For Elementor builds, do not randomly set font values per widget. Define repeated styles globally first.

## Color and Theme Guidance

Use a cohesive color system.

A good color system usually includes:

1. Primary brand color.
2. Secondary brand color.
3. Main background color.
4. Alternate background color.
5. Main text color.
6. Muted text color.
7. Accent color.
8. Button color.
9. Border color.
10. Optional highlight color.

Rules:

1. Dominant colors should be controlled.
2. Accent colors should be purposeful.
3. Do not use too many equal-strength colors.
4. Use contrast properly.
5. Match the reference and brand identity.
6. Add repeated colors to global styles.

## Layout and Composition Guidance

Avoid repetitive layouts.

Use layout variety where appropriate:

1. Split hero.
2. Centered hero.
3. Full background hero.
4. Editorial asymmetric layout.
5. Text-first content section.
6. Image-first content section.
7. Card grid.
8. Stats row.
9. Process timeline.
10. Large quote/testimonial section.
11. CTA banner.
12. Logo strip.
13. Feature comparison.
14. Layered visual stage.

Do not use the same layout rhythm for every section.

## Spatial Design Rules

Spacing is part of the design.

Use:

1. Generous whitespace for premium layouts.
2. Controlled density for campaign or magazine-style layouts.
3. Clear relationship between headings, text, and CTAs.
4. Larger section gaps for major topic changes.
5. Smaller gaps for tightly related content.
6. Consistent card spacing.
7. Consistent container width.

Do not cram content into a section just to match a screenshot. Recreate the visual hierarchy cleanly.

## Visual Details

Use details only when they serve the design.

Possible details:

1. Subtle texture.
2. Fine border lines.
3. Soft shadow.
4. Strong editorial image crop.
5. Large typographic treatment.
6. Asymmetrical layout.
7. Controlled accent shape.
8. Background image overlay.
9. Brand pattern.
10. Section divider.
11. Hover state.
12. Motion cue.

Avoid decorative effects that do not support the brand or content.

## Motion Rules

Motion should support clarity, not distract.

For normal Elementor builds:

1. Use light entrance animation only if it fits the reference.
2. Keep motion subtle.
3. Do not animate every widget.
4. Avoid heavy JavaScript animation unless explicitly requested.
5. Prefer Elementor-native motion effects where safe.
6. Respect performance.

For custom frontend builds outside Elementor, motion may be more advanced if the project calls for it.

## Scroll-Driven Website Rules

Only use this section when the task explicitly asks for a scroll-driven or animated website.

For scroll-driven sites:

1. Use large typography.
2. Avoid generic cards and boxes.
3. Let text sit confidently on the section background.
4. Use varied section layouts.
5. Use color zones.
6. Use at least three layout patterns.
7. Use different entrance animations per section.
8. Use pinned sections only when technically appropriate.
9. Use horizontal text movement only when it supports the story.
10. Keep performance and accessibility in mind.

Do not apply scroll-driven animation rules to normal Elementor pages unless requested.

## Image Direction

Images should feel selected, not random.

Before placing images:

1. Identify the image role.
2. Decide whether it is emotional, informational, product-focused, trust-building, or decorative.
3. Crop intentionally.
4. Keep aspect ratios clean.
5. Use consistent radius or no radius based on the design style.
6. Compress and optimize.
7. Avoid using many unrelated image styles on one page.

## Component Quality Rules

For any section or component:

1. It should have one clear job.
2. It should have a clear hierarchy.
3. It should feel connected to the brand.
4. It should not look like a default template.
5. It should be responsive.
6. It should be editable.
7. It should use global styles where possible.
8. It should avoid unnecessary complexity.

## Design Adaptation Process

When a reference is provided:

1. Identify the design language.
2. Identify the layout structure.
3. Identify typography rhythm.
4. Identify color system.
5. Identify image treatment.
6. Identify spacing scale.
7. Identify button style.
8. Identify repeated patterns.
9. Identify responsive assumptions.
10. Recreate the system, not just the surface.

## Output Format

When using this skill, respond with:

```md
## Design Direction

State the chosen visual direction and why it fits.

## Reference Observations

List the key visual rules from the reference.

## Layout Strategy

Explain the section or page structure.

## Global Styles Needed

List colors, typography, buttons, spacing, borders, radius, and reusable effects.

## Elementor Compatibility Notes

Explain how the design stays editable in Elementor.

## Responsive Strategy

Explain desktop, tablet, and mobile behavior.

## Quality Risks

Mention anything that could become too generic, too heavy, too hard to edit, or too different from the reference.
